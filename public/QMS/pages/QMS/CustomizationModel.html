Process Model for Customization Projects
ISO Clause: 7.1 PM07 Version: 3.00 Date: 17th May 2010

1.0 Introduction

This process model is suggested for projects having the following history and attributes;

1. Code base of the product have been acquired from internal or external sources

2. Management or the customer has approved changes to the existing code base

2.0 Lifecycle

The product needs modifications, additions and removals of the features. These modifications have been requested or agreed upon with the customer and other relevant stakeholders. At this stage the Customization Project lifecycle starts.

The project will go through the following phases.

1. Inception

2. Customization

3. Pilot Deployment

4. Pilot Deployment Acceptance

5. Data Migration

6. Production deployment and signoff

3. Pilot Deployment

On completion of customisation of the product, the product is deployed in the pre-production environment at the clientâ€™s environment. The users training (optional) is provided to the end users of the systems. Detailed tasks and schedule of the pilot phase will be defined in the SPMP; if it is not possible to define the tasks during the preparation of the SPMP, a separate pilot plan should be prepared with activities list with resource allocated for each tasks in consultation with the client before starting the pilot phase. The plan should have clear entry and exit criteria for pilot phase.

4. Pilot Deployment Acceptance

Pilot deployment signoff will mark completion of the phase. Signoff will be obtained from the client PM or client representative designated to the project.

5. Data Migration (Optional)

The PM will prepare a plan for migration of the relevant data into the application. The plan will identify the data for migration in consultation with the client or/and as identified in the project scope. The plan will clearly mention what type of data, how much to be migrated, period from which the data needs to be migrated with the schedule of activities and resources responsible for the activities.

6. Production deployment and signoff

After completion of pilot signoff and data migration the application and the database will be deployed on the production environment. The installation and support team will ensure that the application is functioning as expected. Key stakeholders will be involved in release readiness review of the application on the production environment and traceability between the requirements and the application will be established. After meeting the criteria the application will be live in the production environment and production deployment signoff will be obtained.

The project can be closed after the signoff. If support and maintenance is applicable as per the contract, the PM will arrange for necessary resource allocation for these activities. Other resources will be released and project closure will be conducted as per the process defined in the QMS.
