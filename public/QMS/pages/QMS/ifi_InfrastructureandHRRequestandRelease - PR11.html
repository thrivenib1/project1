Infrastructure and Human Resource Request and Release
ISO Clause: 6.3 PR011 Version: 3.00 Date: 17th May 2010

11.0 Introdution

11.1.1 Purpose
The purpose of this procedure is to describe the mechanism for requesting for hardware and software resources and services.

11.1.2 Policy
This process will be applicable for allocation of system and general resources to the employees of the organization in general and the project team members in particular. The project manager will raise the resource request as stated in this process document.

11.1.3 Scope
This procedure will be applicable whenever hardware and software resources are required or for requesting any maintenance service of hardware and software.

11.3 Task Details

11.3.1 Infrastructure Request and Release

11.3.1.1 Raise Request
The requestor can raise the following types of requests:

Request Type Purpose Form to be Used
Hardware To obtain hardware for the project PCC, Infrastructure Request and Release (FRIFR01)
Software To obtain software PCC, Software Request (FRIFR02)
Job For maintenance of hardware and software including setting up of access rights PCC, Job Request (FRIFR03)

11.3.1.2 Approve RequestThe job request with special job types (Machine Movement and Access Rights) requires approval from the PM. All other job types do not require approvals.

10.3.2 Restoration

After taking the backup the system administrator will restore the backup files to verify whether the backup files are retrievable or not. A new backup will be taken if it is not possible to restore from the first backup.

11.3.1.3 Service RequestsThe Infrastructure Team will service the requests as required by the requestor using the System Administration and Management (FP10) procedure.

11.3.1.4 Infrastructure ReleaseThe infrastructure (additional resources allocated to the project) shall be released under the following situations:

  1. At the end of a project.
  2. To be sent to the client.
  3. When not required for the project any more.
  4. On Employee separation.

The infrastructure may be released to:

  1. System Administration
  2. Administration (to be send to the Client)

Infrastructure (Hardware / Software) release shall be done using the form Hardware / Software Release FRIFR04./ PCC

11.3.2 Human resource Request and Release

11.3.2.1 Allocate / Obtain Resources

  • The TMG will allocate human resources based on the Project requirements using PCC. The Human Resource Allocation Request (FRHRR01) will be used by the PM / Function Head for resource requirements for projects or for support functions.
  • If the resources with the required skill set are available within the organizationâ€™s resource pool, (Skill database maintained by the TMG) resources shall be allocated from there.
  • If there are no free or suitable resources available in the resource pool, TMG shall initiate recruitment of necessary resources using Human Resource Hiring Request (FRREC01) The PM will analyze the skill gaps of the resources allocated using the skill database and organize for training as per the Talent Management Procedure FP05 procedure.
  • In case, the resources are added during the course of the project, the PM is responsible to provide On-the-Job Training to the team members and will maintain On-the-Job Training Record (FRHRR03).

11.3.2.2 Release Resources

  • The PM will release human resources as per the project needs through PCC.
  • PM will ensure that the team members update their CV on completion of the project and send their CV to the Personnel Dept. The Personnel Dept will update the skill database for the team members to include the project experience.
  • The person being released before the project closure shall prepare a Hand-over Report (FRHRR02) and give the report to the person taking over the responsibility. This will be approved by the PM as well.

An Employee Clearance (FRHRR04) form is required for the human resources being released under the following cases:

  • Going abroad on a long term assignment (beyond one month).
  • Separation
  • Transfer to other locations
