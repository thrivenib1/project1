10.1 Introduction

10.1.1Purpose

The purpose of this procedure is to define a mechanism for the recruitment function.

10.1.2 Policy

This process is applicable to the System administration team for maintaining the HW and software in the organisation. The same process will be applicable across all locations of the company.

10.1.3 Scope

This procedure will be applicable to all computer hardware and software used in the organization.

10.2 Stakeholders

10.2.1 Responsibilities

10.2.2 Backup and Retrieval

The System Administrator will ensure backup and retrieval of software on the servers as per the Backup, Recovery and Archive (PR10) procedure.

The System Administrator will ensure disaster recovery process and backup restoration documents will be maintained on monthly basis as per FRBAR01.

10.2.3 Maintain Inventory of Hardware and Software

The System Administrator will maintain the inventory of hardware as per the Inventory of Hardware (FRSYA01) The System Administrator will track the equipment requiring the AMC renewals.

The System Administrator will maintain the inventory of software as per the Software Register (FRSYA02).

AMC requirements for the maintenance of hardware, if any, will be identified and controlled by the System Administrator.

10.2.4 Allocate and Maintain Hardware and Software

For Existing Employees:

The System Administrator will process the following requests:

· Hardware Request

· Software Request

· Job Request

The system administrator will ensure the PCs are free of viruses through installing virus scanners in the PCs. If any viruses are reported, the PC will be isolated from the network till the virus is cleared from the PC.

Hardware Request [2]

The PM can raise a request for additional hardware such as PC, server, RAM and disk space using PCC / E-mail / Hardware Request (FRIFR01). The System Administrator is responsible to take action on the request. In case the hardware is not available, the System Administrator will initiate purchase as per the Purchase(FP08)

Software Request

The PM can raise a request for software using PCC/Software Request (FRIFR02). The System Administrator is responsible for installing the software as per the request. In case the software is not available, the System Administrator will initiate purchase as per the Purchase(FP08) procedure. The software issue will be recorded using Software Issue Register (FRSYA03)/ using 'Soft manage'

10.3.1 Job Request

Any employee can report either a hardware or software problem by using PCC / Job request (FR-11C). The System Administrator is responsible to take action on these requests.

10.3.2 For Newly Joined Employees:

Whenever a new employee joins, the Personnel dept. will send an e-mail request to System Administrator to allocate an e-mail id, HW, SW and any other resource required depending on the role of the employee.

10.3.3 Account Creation

The PCC will be used to indicate the creation of the account. Account Creation request will be made by the PM, DH or the reporting head for their team members. System Administrator will ensure that the account is created as per the request made. The user account login name on the system will be same as email ID. The software required for Installation should be filled in the form.

The System Admin will maintain the User Account Creation (FR62G).

10.3.4 Account Deletion

The PCC will be used to request for deletion of the account.

Deletion of account will be initiated by the PM or the reporting head of the employee, when the employee is moved out of a project or assignment. Before releasing the System from the project, the Project Manager will ensure:

· To take all necessary project work related documents, source code and any other backup from the employee in case Project Repository does not exist on a repository server.

· Check for the correctness of the provided backup.

· The System Admin will maintain the User Account Deletion (FR62G).

10.3.5 Password Policy

· For better security of the password, all users will create alphanumeric passwords of length eight at least for their respective login account.

· The password list will be updated every 90 days and kept with the Head Engineering as well as a copy with the HRD.

· The password should not be repeated.

10.3.6 Privilege Password

Privilege passwords can be created based on specific request from the PM. The PM can log the request through PCC. The system administrator will create the privilege password for the user on the advice of the PM.

10.3.7 Guest Account

Based on the outcome of the interview(s) an offer of appointment will be given to the selected candidate. The following documents have to be provided as annexure to the offer of appointment given to the candidate:

The system administrator will ensure that by default the guest account in all the PCs is disabled. The guest account will be enabled only on specific request of the PM. The PM can log the request through PCC.

10.3.8 Patch Updates

· OS update patches will be downloaded from the authenticated site by the system admin team regularly.

· System admin team will ensure that automatic update is enabled on all systems.

10.3.9 Preventive Maintenance

The system admin team will carry out periodic preventive maintenance work of the workstations. The System Admin will prepare a preventive maintenance plan covering all the computers in the organisation once in every quarter. The System Admin will maintain the Preventive Maintenance Record FRSYA06.

Preventive maintenance will include:

· Running anti-virus software

· Delete cookies (Temp cookies)

· Delete Temp files

· Delete Temporary Internet Files

10.3.10 Release Hardware and Software

The PM will release the hardware and software resources used by the project using Hardware /Software Release (FRIFR04).

PM will release the Hardware and software using PCC for the project members.

The system administrator will acquire the released resources for re-allocation. The system administrator will uninstall the released software prior to reallocation.

The system Admin to ensure following on release of Project PC.

· Remove all project related software installations.

· Create a Login with the name Integra with Local User Rights and make the Pc available for free pool

· Format the System if required.

10.3.11 Account Change Request for Systems

All system and software related change request would be made using form, Change Request for Software And Hardware . The change request will be made when the following changes are required

· Change in Software Requirement. Installation/Removal or Movement of License.

· Change of System due to configuration issue.

· Change in User Account Type/Permission/Policy.

· Change in Security policy

· Configurations of a mail ID other than the end users default mail ID.

10.3.12 OS hardening

System Administrator shall update the system with latest patch and antivirus definition before allotment of the system.

10.3.13 Report Infrastructure Status

Monthly

System Administrator will prepare an Infrastructure Monthly Status Report (FRSYA05) and submit it to Head–Engineering. The Head-Engineering will forward the status report to the director for discussion in the operational review meeting as stated in the Management Review procedure

10.3.14 Tracking Downtime

The System Administrator will monitor the Network and Server-downtime by using the Equipment Downtime Log (FRSYA04) and take suitable corrective action.

Logical Security

The Head-Engineering will ensure that the super-user / root password for servers and routers are changed periodically. The Head-Engineering will maintain a changed password list. The Head-Engineering will maintain a hard copy of the router's configuration.

10.3.15 Conduct Software Audits

Through software audits the System Administrator will ensure the following:

· No un-licensed or unnecessary software is installed on any PC

· PCs are free of viruses

The System Administrator will audit workstations once in three months. The result of the audit will be recorded using Software Audit Report (FRSEC01).

If any usage of unauthorized software is found, the same would be deleted and the Department Head would take appropriate action against the employee.

However, employees can use any freeware available on the corporate Intranet.

Software audits will be done on a random sampling basis.

10.3.16 Maintain Records

The following folders will be maintained:

· Backup Log

· Equipment Downtime Log

· Job Requests

· Hardware Requests

· Software Requests

Head -Engineering will maintain the following documents:

· Router Configuration

· Network Diagram

· Inventory of Hardware

· Infrastructure Monthly Status Report

· Software Audit Report

10.3.17 MLD and MLR

The System Admin will maintain MLD (FRPDM01) and MLR (FRPDM02) for the function.

10.3.18 Metrics for Infrastructure Management

The Metrics pertaining to the function will be collected as per Metric Plan and analyzed (monthly) for corrective / preventive action. Internal customer satisfaction index can be calculated once in a year by pertaining records from HR(ESS).

10.3.19 Retention and Disposition of Records

· The owner of the record will maintain the records for a period of one year or as required by the contract if applicable.

· In January every year, each Function Head will identify all records pertaining to the function which are more than a year old. All such records will be packed and suitably labelled and handed over to the General Admin In-Charge for storage.

· In January every year, the General Admin In-Charge will identify all records, which are more than three years old and will destroy them as applicable.

· The General Admin In-Charge will maintain the list of the stored item using Archival Register FRARC01.

Inventory of hardware and software register will be maintained in PCC. In case of non-availability of PCC, the respective forms can be used as a backup.

[2]

Hardware and software requests will be maintained in PCC. In case of non-availability of PCC, the respective forms can be used as a substitute.
