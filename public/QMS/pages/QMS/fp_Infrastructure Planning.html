Infrastructure Planning
ISO Clause: 6.3 FP16 Version: 3.60 Date: 30th March 2015

16.0 Introduction

16.1.1. Purpose

The purpose of this procedure is to define a mechanism for infrastructure planning.

16.1.2. Policy

This process is applicable for planning infrastructure requirements for the company, which includes yearly requirements of: general infrastructure, hardware, software, network bandwidth, power, UPS, safety equipments, etc.... of the organisation for all the locations.

16.1.3. Scope

This procedure will be applicable to all computer hardware and software used in the organization.

16.2 Stakeholders

16.2.1 Responsibilities

Sl. No. Role Responsibility Authority
1. K Sridhar Obtain Requirements, Communication Links Plan, Hardware Plan, Software Plan, Prepare Infrastructure Plan
2. Managing Director Review Infrastructure Plan

16.3 Task Details

16.3.1 Obtain Requirements

The K Sridhar will obtain the following requirements:

Requirements/Input Responsible
Recruitment Plan TM
Project Requirements PMs
Support Requirements Functional Heads
Business Plan Director

16.3.2 Communication Infrastructure

International Links

The K Sridhar will determine the required capacity based on the number of people. Typically, the capacity planning is based on an assumption of 1 kbps bandwidth per employee. This includes bandwidth on both Internet and dedicated links.

Depending on the new business opportunities, the K Sridhar will determine the following:

  • New communication link requirements
  • Existing communication links' upgrade requirements

Domestic Links

The K Sridhar will incrementally strengthen the domestic communication links based on voice and data communication traffic, which will enable sharing of resources.

16.3.3 Hardware Plan

The K Sridhar will plan the hardware requirements based on the following broad assumptions:

Category Thumb Rule
PCs One per employee
Backup Server One - to be used by all projects.
Common Servers Mail Server, Firewall/Proxy, Web Server, Departmental Servers, RAS Server, Fileserver
Printers One per 50 employees
Switches Based on the seating capacity
Modems for DOT Links 2-4 per link
Modems for Dial-ups 1 per 100 employees
Upgrades 30% of value of new purchases
Project specific purchases A percentage of the total project revenue based on previous year

16.3.4 Software Plan

The K Sridhar will plan the software requirements based on the following broad assumptions:

Category Thumb Rule
Client Software One license per employee
Mail Server One per development center
Project-specific A percentage of the total project revenue based on previous year
Departmental Software As per departmental requirement

16.3.5 Prepare Infrastructure Plan

The K Sridhar will compile the communication, hardware and software plans into an infrastructure plan, which specifies the quarterly expenditure plan.

16.3.6 Review Infrastructure Plan

The Managing Director will review and approve the infrastructure plan. The approved items will be procured as per the Purchase (FP08) procedure.
