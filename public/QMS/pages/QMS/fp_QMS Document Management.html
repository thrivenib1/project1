QMS Document Management
ISO Clause: 4.2.3 FP14 Version : 3.60 Date : 30th March 2015

14.1 Introduction

14.1.1 Purpose

The purpose of this procedure is to describe the mechanism for QMS document control.

14.1.2 Policy

The process is applicable for management of all documentation relating to the QMS.

14.1.3 Scope

This procedure will be applicable to all QMS documents generated to define software project related process and the support function procedures of Integra.

14.3.1.1 Document States

Any QMS document will be in one of the following states:

  • Draft: An unapproved document will be in "Draft" state.
  • Controlled: An approved document will be in "Controlled" state.
  • Un-Controlled: An approved document that is issued to an external entity will not be controlled.
  • Superseded:Whenever a new version of the document is approved and released, the old version will be in "Superseded" state.

14.3.1.2 Version Numbering
All the documents will have a version number specified in the following format:

Version x.yy where "x" represents the issue status number and "yy" represents the revision number.

14.3.1.3 Revision History

The documents revision history is maintained as specified in the following format:

Document Name Document ID Version Brief Description of change QMS change request Effective date

14.3.2 Maintain the Master List of Documents

The MR will prepare and maintain a Master List of Documents (FRPDM01).

14.3.3 Approve Documents

The draft of first version of the documents will be reviewed and approved as per the Verification procedure (PR-07).

Review and approval of document will be carried out as explained in section 14.2.1 of this document. All documents will be approved prior to use.

For major changes (where the issue status number is incremented), the entire set of documents will be reviewed using Verification procedure (PR-07).

14.3.4 Updates of QMS documents

All updates of QMS documents will be reviewed by the respective authority responsible for the document as defined in section 14.2.1. For the updates it is not required to maintain the review record. The revision section in the manual will be updated upon any update of the particular section of the manual.

14.3.5 Document Storage and Archival

The MR will maintain a master soft copy of the QMS documents in CVS. The controlled soft copies will be deployed in the Intranet for access by all employees.

14.3.6 Issue Control

The MR will issue control copies of the QMS documents in PCC.

Electronic Media

All Documents in electronic media will be controlled as follows:

a. The version control will be ensured using CVS or SVN

b. Write access to the Controlled version only for the MR.

c. The Controlled copies will be deployed on Intranet. Read access will be provided to all.

Control of QMS documents

Version control of the QMS documents will be ensured by maintaining a folder (Library) structure as follows:

.\QMS_Docs

01_Development

02_Baseline

03_Superceded

Library Description
Development TO store the QMS documents that are under development or modification
Baseline To store all the current controlled QMS document that are in use
Superseded To store the superseded QMS documents

Printed Copies

The printed copies are, by default, "Uncontrolled" unless they are stamped as "Master" or "Controlled copy".

14.3.7 Document Modification

Modification to the QMS documents will be based on any of the following inputs:

  1. Change request from users (through email)
  2. Action item from Management Review Meeting
  3. Audit Findings
  4. Customer feedback OR
  5. Any other feedback

    6. All requests for change (sl.no b to e) to the QMS documents will be recorded by SQA using the PCC through email. The SQA, MR, will review and approve the change requests.

    Once the changes to the document is made and approved, MR will ensure that the Revision History is updated for the changes.

    All new QMS documents will be created using the following document templates:

Sl No. Volume Template
1 Templates Standard Document Template (TPOPD02).
2 Procedure Manual Procedure Template (TPOPD01).
3 Checklist Checklists Template (TPOPD03).

14.3.8 Issuing Revisions

Electronic Media

The "Controlled" soft copy of the documents deployed on the Intranet will be updated with the new version of the documents.

14.3.9 Deployment of QMS on Intranet

The MR will request the System Administrator for deployment of the modified document(s).

The QMS document(s) will have read-access to all the users. The MR will update the QMS Master List of Documents (FRPDM01) .

and announce the deployment of the new QMS version.

14.3.10 Documents of external origin

All documents of external origin will be controlled. All customer supplied documents and material will be controlled as defined in the Control of Customer Supplied Material procedure (PR-12).

14.3.11 Retention and Disposition of Records

  • The owner of the record will maintain the records for a period of one year or as required by the contract if applicable.
  • In January every year, each Function Head will identify all records pertaining to the function which are more than a year old. All such records will be packed and suitably labelled and handed over to the General Admin In-Charge for storage.
  • In January every year, the General Admin In-Charge will identify all records, which are more than three years old and will destroy them as applicable.
  • The General Admin In-Charge will maintain the list of the stored item using Archival Register FRARC01.
