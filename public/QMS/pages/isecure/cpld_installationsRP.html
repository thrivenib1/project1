13.4.1. General

The policy for information security defined below has been approved by management for publication and communication to employees along with relevant external parties.

13.4.2. Policy on installation of Software on Operating Platform

13.4.2.1. Change management process of the organization must be followed.

13.4.2.2. A rollback strategy must be in place before installation of software or changes are implemented in existing software on operating platforms.

13.4.2.3. Only the approved software's of the company can be installed in any operating platform, by the IT team only.

13.4.2.4. The updating of the operational software, applications and program libraries must only be performed by trained administrators upon appropriate management authorization.

13.4.2.5. Operational systems must only hold approved executable code and operating system software must only be implemented after extensive and successful testing.

13.4.2.6. The tests must cover usability, security, effects on other systems, interaction support and user friendliness and such test must be carried out on separate systems.

13.4.2.7. Physical or logical access must only be given to suppliers for support purposes when necessary and with management approval.

13.4.2.8. The supplier's activities must be monitored.

13.4.3. Policy on restrictions on software installation

13.4.3.1. Only software's identified in the organization as approved software's are allowed to be installed in the organization.

13.4.3.2. IT department will obtain and track the licenses, test new software for conflict and compatibility, and perform the installation

13.4.3.3. The principle of least privilege for software installation amongst users must be applied.

13.4.3.4. The IT department of the organization must only install the software's, unless privilege rights are granted for the user, through a formal risk assessment.

13.4.3.5. When such software's which are installed through privileged rights, IT department maintains the list of software's that can be installed by such privilege users.

13.4.3.6. Such systems, which have software installation permission by the user must be reviewed by the IT department on fortnightly basis to control and monitor the nature and type of software's installed by the user.

13.4.3.7. Software audits must be conducted as per QMS procedure FP10. System Administration and Management.
