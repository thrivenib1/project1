1.     Initialization

To improve the application load time, place the the <script> tag including the angular.js and related angular scripts at the bottom of the page. The HTML loading will not be blocked by angular.js and related scripts loading.

2.     Expressions

Complex javascript code should be added as a method to the $scope in the controller and then called from the view. This process is different from adding the complex Javascript code as Angular expressions in the view.

3.     Controllers

In the real time applications, avoid creating controllers in global scope. Rather, use “.Ctrl” method of the Angular Module to work with $scope object. A sample code is given below illustrating this point:

The controllers should only be used to setup initial state of the $scope object and add one or more behaviour to this object. One should avoid using controllers to do some of the following actions:

·        Manipulate DOM, 

·        Format input, 

·        Filter output, 

·        Share code or state across different controllers

·        Manage the life-cycle of other components (for example, to create service instances)

A controller should contain only the business logic needed for a single view. The functionality should be moved to services and these services should be injected into the controllers using dependency injection.

4.     Directives

·        It is preferred to use the dash-delimited format (e.g. ng-model for ngModel). While working with an HTML validating tool, as an alternative use the data-prefixed version (e.g. data-ng-model for ng Model).

·        It is preferred to use the directives via tag name and attributes over comment and class names. This helps in determining which directives is suitable for a given element.

·        While creating directives, it is recommended to prefix your own directive names to avoid collisions with future standard.

5.     Template

With large templates (HTML content with Angular directives) within an HTML file, it is recommended to create separate HTML files and load it with the templateUrl option.

6.     Dependency Injection

It is recommended to inject the dependencies by passing the dependency to the constructor function rather than using any other different manner. This way, the responsibility of creating the dependency object lies with other objects or function. This is a direct approach for those who have worked with one or more dependency injection framework in the past. However, this could be a useful information for the beginners.

Following are the different ways of doing dependency injection in Angular:

·        Create it using the new operator.

·        Look for it in a well-known place, also known as a global singleton.

·        Ask a registry (also known as service registry) for it.

7.     Angular UI-Router

Include angular-ui-router.js (or angular-ui-router.min.js ) in your index.html, after including Angular itself (For Component users: ignore this step).

Add 'ui.router' to your main module's list of dependencies (For Component users: replace 'ui.router' with require('angular-ui-router'))

When you're done, your setup should look similar to the following:

<!doctype html>

<html ng-app="myApp">

<head>

    <script src="//ajax.googleapis.com/ajax/libs/angularjs/1.5.8/angular.min.js"></script> 

    <script src="js/angular-ui-router.min.js"></script> 

    <script>

        var myApp = angular.module('myApp', ['ui.router']);

        // For Component users, it should look like this: 

        // var myApp = angular.module('myApp', [require('angular-ui-router')]); 

    </script> 

    ...

</head>

<body>

    ...

</body>

</html>

8.     Nested States & Views

The majority of UI-Router's power is in its ability to nest states & views.

(1)  Follow the setup instructions detailed above.

(2) Add a ui-view  directive to the  <body /> of your application.

 

<!-- index.html -->

<body>

    <div ui-view></div>

    <!-- We'll also add some navigation: -->

    <a ui-sref="state1">State 1</a>

    <a ui-sref="state2">State 2</a>

</body>

 

(3) You will notice that some links are added with ui-sref  directives. In addition to managing state transitions, this directive auto-generates the  href attribute of the <a /> element it is attached to, if the corresponding state has a URL. Then some templates will be added. These will plug into the ui-view within index.html. Notice that the UI router will have their own  ui-view. This is the key to nesting states and views.

 

<!-- partials/state1.html -->

<h1>State 1</h1>

<hr/>

<a ui-sref="state1.list">Show List</a>

<div ui-view></div>

<!-- partials/state2.html -->

<h1>State 2</h1>

<hr/>

<a ui-sref="state2.list">Show List</a>

<div ui-view></div>

 

(4) Some child templates will be added. These will get plugged into the ui-view of their parent state templates.

 

<!-- partials/state1.list.html -->

<h3>List of State 1 Items</h3>

<ul>

  <li ng-repeat="item in items">{{ item }}</li>

</ul>

<!-- partials/state2.list.html -->

<h3>List of State 2 Things</h3>

<ul>

  <li ng-repeat="thing in things">{{ thing }}</li>

</ul>

 

(5) Finally, we will wire it up with $stateProvider . Set up your states in the module config, as in the following:

 

 

app.config(function($stateProvider, $urlRouterProvider,$httpProvider) {   

 

      $stateProvider.state('About', {

          url: "/About",

          title : "About",

           views : {

                                              "viewMenuBar"  : partials.menuBar,

                                              "viewFooter@About"  : partials.footer,

                                              'RightSideBar@About': partials.sidebar,

                    "viewBody" : {

                        templateUrl: "views/templeats/About .html",

                        controller : "AboutCtrl"

                    }

                },

        })

   });

 

9.     Partial View Inside page

We used Partial view inside the respective page using partial variable.

 

Example:-

      var partials = {

                menuBar : {

                  templateUrl : 'views/partials/header.html',

                    controller : ''

                },

                footer : {

                  templateUrl : 'views/partials/footer.html',

                    controller : ''

                },

                 rightSidebar: {

                        templateUrl: 'views/partials/sideBar.html',

                        controller: 'sidebarCtrl'

                }

            };
