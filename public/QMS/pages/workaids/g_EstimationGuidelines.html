Estimation Guidelines

GLPPL01

Version: 3.00

Date: 17th May 2010

 

1      Purpose

The purpose of this document is to describe the guidelines to be used for effort estimation during Proposal Preparation and subsequent re-estimation during Analysis and Design phase.

2      Introduction

The process of creating estimates consists of the following steps:

�        Identify the development/target environment.

�        Estimate the size of the software product using Work Breakdown Structure or other estimation techniques.

�        Estimate the effort.

�        Estimate the schedule.

�        Identify critical dependencies from customer/other groups.

�        Identify staffing requirements.

�        Identify critical resources (tools required, when required).

�        List down the previous project data referred to.

The Estimation Worksheet Template (TPPPL02) or Object Point Estimation Template (TPPPL02A) is to be used for estimation.

3      Identify Development/Target Environment

Identify the environment selected for the project.

4      Estimate Size Using Estimation Techniques

Although estimating is as much an art as it is science. Estimation lays a foundation for all planning activities and project planning provides a roadmap for successful software engineering, a good estimate is the first step towards a successful project.

The estimation technique to be used depends on the nature of the project. To achieve a reliable estimation, base the estimates on similar projects executed in the past, use relatively simple decomposition using work break down technique. �        classify components based on type (function) and

The purpose of complexity characterization is primarily for estimation. This measure can be used at the time of estimation, when little is known about the program's interface or logic, but considerable information is available on their functionality or their specification. 4.2  Maintenance Load Estimation

The maintenance effort estimation could be based on one or more of the following:

�        Number of requests estimated per month

�        Size of the application

�        Number of resources in the current maintenance team

�        Backlog and nature of maintenance activities to be done

�        Typically a core team size is estimated and peak load may be handled through additional staffing in consultation with the customer

6      Project Effort

The Project Effort is calculated using the following relation:

Project Effort (in person month) = a (Size in KLOC)b

Where, a and b are two constants whose value is found using regression analysis of past data from OPD.

Based on the effort so obtained the schedule for the project is found. The Estimation for the entire life cycle of the project is arrived using the estimation worksheet

templates TPPPL02 OR Object Point Estimation Template (TPPPL02A)

In addition to the above method of calculating efforts, the estimates could be obtained from more than one source (Person) or method (WBS or FP).

7      Re-estimation

The project should be re-estimated after completion of the analysis and design phase. The re-estimated effort, schedule and cost should be shared with all project stakeholders and the SPMP should be updated with the re-estimated values.
