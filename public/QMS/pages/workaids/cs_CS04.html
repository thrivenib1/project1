4.3.6 Constants

The body of constant names should be in UPPERCASE with underscores(_) between words.

Example: gstrNEW_LINE � New Line Character String { Global to entire application )

Although standard Visual Basic constants do not include data type and scope information, prefixes g, and m can be very useful in understanding the value and scope of a constant. For constant names, follow the same rules as variables.

4.4 Programming Standards

4.4.1     Error Handling

An error that occurs under the following conditions, should be logged in the Event Log File

�         Disk Operations

�         Network Operations

�         Console Operations

All programs with a user interface should report the error with a message box. The message box should contain

�         Name of the process reporting the error

�         Error Message(with the appropriate icon)

�         Suggested next steps (including calling for assistance)

There should be an Exit Sub before the start of the error handler.

4.5 Source Code Standards

4.5.1Commenting

Every source file should have Source file header. A Source file header should contain:

�         The copyright and proprietary information

�         The source file�s filename

�         The name of the unit the source file belongs to

�         The original author of the file

�         Date of Creation

�         Functional description of the source file

�         Version Number

Every change in the source file should have a change header. A change header should contain:

�         Change Description

�         Author

�         Review Period

Example:

'=========================================================================

�Copyright (c) 1998-99 All Aero

�All Rights Reserved

�This work contains trade secrets and confidential material of Allied Aerospace, and its use of �disclosure in whole or

in part without express written permission of All Aero is Prohibited.

�File Name : frmOrder [ SACMOrder.frm ]

�Author : Integra Micro Systems Pvt. Ltd

�Date of Creation : 10 February 1999

�Description : To enter new order details

�Version Number : 1.0

'=========================================================================

Every function/procedure of a source file should have a header. The name of the function/procedure should start with an uppercase alphabet.

A function/procedure header should contain:

�         The name of the function or procedure

�         Values which are passed to the function

�         Values which are returned from the function

Example:

'=========================================================================

' Procedure Name : Sacm_P_AssembleNote

' Procedure description : This Procedure is used to make a string of information depending on the Selected values in

combo box and displays in the free text box

' Values Passed : None

' Values Returned : None'=========================================================================

The "hard-coded" constant values should have comments for clarity

4.5.2 Source Code Complexity

The source code should be implemented with lowest number of loops and branches possible to reduce complexity.

4.5.3     Unit Structure

�         Individual functions or procedures of a unit should avoid any functional overlap or redundancy of functionality while remaining independent.

�         Values passed to and returned from the functions should use simple variables or strings in preference to structures.

�         The code should be well indented with tabs.

�         All constants should be named when referenced in source code. That is, numeric values should never reference directly within source code.

�         When using abbreviations, make sure they are consistent throughout the application

4.6   Visual Basic Guidelines

�         Use Option Explicit

�         Use an ampersand (&) when concatenating strings

�         Use the plus sign when working with numerical values

�         Declare all the global variables in a single module (BAS file).

�         Any confirmation from the user should be accepted in a message box

�         User should not be able to modify the contents of the text boxes that are used to display information

�         Wherever an "OK" button is present, make the same as the default one.

�         In cases such as canceling an operation, get confirmation from the user.

�         The following shortcut keys shall be used for these options:

�         OK : "O" Cancel : "C" Update : "U" Delete : "D" Exit : "E"

�         Gray out the display-only fields so that the user can easily differentiate between input/output entry fields and display-only fields.

�         Display Hourglass as an indication to the user response before performing data if the process time is going to exceed 2 or 3 seconds

�         Select MS Sans Serif � Regular � 8 font for the form as well as controls.

�         Restrict the user from exiting the application from more than one place.

�         To release whole memory used by a form module , set form_name = Nothing after the form unload statement

Example : Unload form1

Set form1 = Nothing

�         Use dynamic arrays instead of static arrays, because dynamic arrays can be resized at any time and hence saving memory. Use Redim statement to declare dynamic arrays.

�         Only one variable should be declared in a line. While declaring, specify the type explicitly for each variable. Otherwise VB will create Variant type.

�         Before calling a procedure/function, the parameters to be passed have to be appropriately initialized.

�         All controls/Objects that are not currently active should be grayed out (disabled).

�         There should not be more than 80 characters per line. The code should be properly indented.

�         Typically the date fields should be in the �mm-dd-yyyy� format.Y2K compliance has to be taken care while displaying, storing, processing and printing data.

4.7   GUI Design

4.7.1     Form

�         The Caption property of the Form should have appropriate Purpose and Business Description.

�         The BorderStyle property of the Form should not be Sizable.

�         The StartUpPosition property should be set to Center Screen.

�         The Name property should be appropriate with reference to the Purpose and Business Description of the Form, it should always start with frm (formname) incase the form is a standalone form and with mdi (formname) in case the form is a child form.

�         The Background color should be consistent with other objects on the form.

�         Control box property of the Form should be set to False.

4.7.2     Text Box

�         The Name property should be appropriate with reference to the contents of the text box, It should always begin with txt (Text Box name) e.g. if the text box is going to contain the Clients Name then it�s Name should be txtClientName.

�         The BorderStyle property of the text box should be Fixedsingle

�         The Maxlength property of the text box should be set appropriately.

�         If the Text Box is not editable then its Locked property should be set to true.

�         The Text property should be set to "" at design time.

�         The Height property should be set to 300 twips.

4.7.3     Command Button

�         The Name property should be appropriate with reference to the Action of the command button, It should always begin with cmd (command button name) e.g. cmdOk.

�         The Caption property should be appropriate with reference to the Action of the command button.

�         The Height property should be set to 350 twips.

4.7.4     Label

�         The Name property should be appropriate with reference to what the label will be used to display, It should always begin with lbl (label name) e.g. lblClientName,

�         The Caption property should be appropriate with reference to what the label will be used to display.

�         The Height property should be set to 300 twips.

�         The Autosize property of the label should be true.

�         The label can have borders.

�         The Background color should be the same as the Background color of the Form.

4.7.5     Masked Edit Box

�         MaskEdit Box shall be used for all Date Text Boxes.

�         The Mask property should be set to "##/##/####" for date fields.

�         The Name property should be set appropriately with reference to the contents of the box and it should begin with me (MaskEditBoxName).

�         The Height property should be set to 300 twips.

�         The Width property should be set appropriately (preferably to 975 twips).

�         The AllowPrompt property should be set to false.

�         The PromptInclude property should be set to false.

�         The ClipMode property should be set to mskExcludeLiterals (1).

�         The PromptChar property should be set to "_".

�         The AutoTab property should be set to false (Default).

4.7.6     General

�         All the controls on the form should be left aligned under one another.

�         The Left property of all the controls, which appear on the Left-hand side of the screen (Label�s) should be 120 twips.

�         A gap of 1 line should be left between each control while placing them vertically.

�         All the mandatory fields on the form should be colour coded.

�         The tab order for navigating through the screen should be proper (left to right and top to bottom).
